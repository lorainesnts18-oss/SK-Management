# SK Management System

SK Management is a full-stack web application for Sangguniang Kabataan workflow management. It includes login, dashboard updates, event tracking, calendar views, organization chart management, and fund transaction tracking.

## 1) Project Overview

This project combines:

- A Node.js + Express backend
- A SQLite database
- Multi-page HTML/CSS/JavaScript frontend modules (Dashboard, Events, Calendar, Org Chart, Funds, Login)

Main responsibilities:

- User authentication (demo account)
- Notes and reminders management
- Event lifecycle management (create, edit, complete, clear done)
- Calendar visualization with holiday support
- Organizational chart CRUD operations
- Fund transaction and balance tracking with optional receipt images

## 2) Tech Stack

- Runtime: Node.js
- Backend: Express 5
- Database: SQLite3
- Frontend: Vanilla HTML, CSS, JavaScript (no frontend framework)
- Storage (session-like state on client): localStorage

Dependencies from package.json:

- express
- sqlite3

## 3) Folder-by-Folder Documentation

## Root Folder

### index.html

- Entry page that checks localStorage key skIsLoggedIn.
- Redirect behavior:
  - Logged in: Sections/Dashboard/dashboard.html
  - Not logged in: Sections/Login/login.html
- Contains fallback UI if redirect does not happen.

### server.js

- Main backend server and API implementation.
- Creates/opens SQLite DB at Database/sk_management.db.
- Initializes tables and seed data on startup.
- Serves static files from project root.
- Exposes REST endpoints under /api.

### package.json

- Defines app metadata, dependencies, and scripts.
- Start script: node server.js && start index.html
  - Important: because node server.js is long-running, open the app manually in browser if needed.

### package-lock.json

- Auto-generated lockfile for deterministic dependency installation.

### node_modules

- Installed dependencies (generated by npm install).

## Database/

### sk_management.db

- SQLite database file created/managed by server.js.
- Stores users, notes, events, fund transactions, and org chart positions.

## Image/

### Logo.png

- Main logo used across login and section pages.

## Sections/

This folder contains all UI modules. Each module is split into:

- page HTML
- page JavaScript logic
- page CSS styling

### Sections/Login/

#### login.html

- Login form with username/password fields.
- Shows demo credential hint: admin / admin123.

#### script.js

- Calls POST /api/login.
- On success sets:
  - localStorage skIsLoggedIn = true
  - localStorage skUsername = logged in username
- Redirects to dashboard.
- Prevents file:// mode usage and instructs running via local server.

#### styles.css

- Login page styling and layout.

### Sections/Dashboard/

#### dashboard.html

- Main dashboard shell with sidebar navigation.
- Sections for:
  - Updates
  - Changes
  - Reminders
  - Quick note form
- Includes logout confirmation overlay.

#### script.js

- Enforces auth check via localStorage.
- Loads notes and events from API:
  - GET /api/notes
  - GET /api/events
- Adds and deletes notes:
  - POST /api/notes
  - DELETE /api/notes/:id
  - DELETE /api/notes (reset all)
- Calculates upcoming-event reminders, including birthday handling.
- Updates dashboard counters and refreshes periodically.

#### styles.css

- Dashboard page styling, cards, sidebar, responsive mobile behavior.

### Sections/Events/

#### events.html

- Event management page.
- Displays:
  - Upcoming events table
  - Completed events table
- Includes modal form for add/edit event.

#### script.js

- Enforces auth check.
- CRUD and status actions for events:
  - GET /api/events
  - POST /api/events
  - PUT /api/events/:id
  - PATCH /api/events/:id/complete
  - DELETE /api/events/done
- Supports custom event type when Other is selected.
- Special date comparison logic for birthdays.

#### styles.css

- Events page styling for tables, modals, controls, and responsive layout.

### Sections/Calendar/

#### calendar.html

- Calendar month grid UI with day selection.
- Right sidebar panel for selected-day event list.

#### script.js

- Enforces auth check.
- Loads events via GET /api/events and renders monthly view.
- Highlights:
  - days with events
  - birthdays by month/day
  - fixed Philippine holidays
  - movable Holy Week holidays (calculated via Easter)
- Supports previous/next month navigation.
- Refreshes data periodically.

#### styles.css

- Calendar styling, event highlights, holiday state, and responsive behavior.

### Sections/OrgChart/

#### orgchart.html

- Organizational chart page with controls for adding rows and positions.
- Includes modals:
  - Add row
  - Add position
  - Edit position

#### script.js

- Enforces auth check.
- Loads org chart rows from GET /api/orgchart.
- Supports management actions:
  - POST /api/orgchart/rows
  - POST /api/orgchart/positions
  - PUT /api/orgchart/positions/:id
  - DELETE /api/orgchart/positions/:id
- Handles optional photo uploads as Base64 data URLs.

#### styles.css

- Org chart node layout, modal styling, and responsive navigation behavior.

### Sections/Funds/

#### funds.html

- Fund tracking interface.
- Displays current balance and transaction history.
- Includes:
  - Add funds modal
  - Deduct funds modal

#### script.js

- Enforces auth check.
- Loads funds state from GET /api/funds.
- Creates transactions via POST /api/funds/transactions.
- Add mode:
  - Handles amount, date, category, and custom category.
- Deduct mode:
  - Handles amount, date, category, details.
  - Requires receipt for most categories except transportation.
  - Converts uploaded receipt to Base64 data URL.
- Recomputes and displays running balance from backend response.

#### styles.css

- Funds page styling for tables, balance display, and transaction modals.

## 4) Database Schema

The backend initializes these tables:

- users
  - id, username, password, full_name
- notes
  - id, type(update/change/reminder), text, created_at
- events
  - id, name, place, date, time, type, notes, done, status, created_at
- fund_transactions
  - id, type(add/deduct), amount, category, details, date, receipt, created_at
- orgchart_positions
  - id, row_index, position_index, title, description, photo, created_at

Seeded defaults include:

- Demo user account
- Starter dashboard notes
- Sample events
- Sample fund transactions
- Starter org chart rows/positions

4. Login using demo account:

   username: admin
   password: admin123

## 7) Environment and Runtime Notes

- Default server port is 3000.
- Override with environment variable PORT.
- Backend serves static files from project root.
- Database file is created automatically if missing.

## 8) Auth and Session Behavior

- Frontend uses localStorage keys:
  - skIsLoggedIn
  - skUsername
- Pages under Sections redirect to Login if not authenticated.
- Logout clears localStorage auth keys and redirects to Login.

## 9) Data Reset and Re-seeding

To reset to initial seeded data:

1. Stop the running server.
2. Delete Database/sk_management.db.
3. Start server again with npm start.

The app will recreate tables and seed default content.

## 10) Known Observations

- Root index.html references Image/SK.png for favicon and fallback image.
- Current image folder includes Image/Logo.png.
- If favicon/logo is missing on the root page, update those references to Logo.png.

## 11) Security Note

- Credentials are stored in plaintext in the SQLite users table.
- This is suitable for demo/local use only.
- For production, use password hashing, secure sessions, CSRF protection, and hardened auth flows.

## 12) Suggested Future Improvements

- Hash passwords using bcrypt.

- Add backend input validation library and centralized schema validation.
- Add automated tests for API and frontend behavior.
